# GitHub Copilot — Anweisungen für Disruptive OS V6.0

## Projektkontext

Du arbeitest an **Disruptive OS V6.0**, einem autonomen NixOS-basierten KI-Betriebssystem namens **Kirobi**.

## Architektur-Überblick

- **2 NixOS-Maschinen**: `kirobi-machine` (Ryzen 9 5900X + RTX 3090) und `home-server` (ZFS-Storage)
- **Deklarativ**: Alle Konfigurationen sind in Nix-Flakes geschrieben — reproduzierbar und versioniert
- **Lokal-First**: KI-Berechnungen laufen auf der RTX 3090, keine Cloud-Abhängigkeit
- **ZFS-Storage**: Pool "tank" (Mirror), Pool "backup" (HDD), private Datasets pro User

## Coding-Richtlinien

### Nix/NixOS
- Verwende immer **Flakes** (kein legacy `nix-channel`)
- Nutze `nixos-unstable` oder stabile Kanäle konsistent — hier: `nixos-24.11`
- Alle Module müssen in `configuration.nix` importiert oder direkt definiert sein
- Secrets via `age`/`agenix` verwalten — niemals Plaintext-Passwörter committen
- Benutze `nixos-hardware` für Hardware-spezifische Optimierungen
- Home-Manager via Flake integrieren

### Python (Kirobi Core)
- Python 3.11+
- Typ-Annotierungen für alle Funktionen (`from __future__ import annotations`)
- Async-first mit `asyncio` wo sinnvoll
- Logging via `structlog` oder Standard-`logging`
- Keine globalen Mutablen States
- Fehlerbehandlung mit spezifischen Exceptions

### Shell-Skripte
- `#!/usr/bin/env bash` als Shebang
- `set -euo pipefail` am Anfang jedes Skripts
- Variablen immer quoten: `"${VAR}"`
- Sprechende Fehlermeldungen mit `echo >&2`

### YAML/Konfiguration
- Kommentare für jeden nicht-offensichtlichen Wert
- Geheimnisse als Platzhalter: `CHANGEME` oder `<SECRET>`

## Sicherheitsrichtlinien

- Keine Secrets im Repository
- Minimale Berechtigungen (Principle of Least Privilege)
- Firewall immer aktiviert
- Tailscale für Remote-Zugriff statt Port-Forwarding

## Was Copilot NICHT tun soll

- Keine Cloud-Dienste vorschlagen (AWS, GCP, Azure) — alles lokal
- Keine `nix-channel`-Befehle — immer Flakes
- Keine Plaintext-Passwörter in Konfigurationsdateien
- Keine unnötigen externen Abhängigkeiten

## Namenskonventionen

| Bereich | Konvention |
|---|---|
| Nix-Attribute | `camelCase` |
| Shell-Variablen | `UPPER_CASE` |
| Python-Klassen | `PascalCase` |
| Python-Funktionen | `snake_case` |
| Dateien | `kebab-case.nix` oder `snake_case.py` |
